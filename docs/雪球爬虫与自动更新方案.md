# 雪球爬虫与自动更新方案

> 本文档整理了雪球爬虫的配置、自动更新机制和使用方法。

---

## 一、项目概述

### 目标
- 自动抓取段永平（雪球昵称：大道无形我有型，ID: 5497652514）的帖子
- 发现帖子中提及的新公司
- 根据帖子内容自动优化网站数据

### 更新频率
- **每周抓取一次**：检测新增帖子
- **每月优化一次**：根据新内容更新网站展示

---

## 二、Cookie 配置

### 2.1 获取 Cookie

1. 用 Chrome 打开 https://xueqiu.com 并登录
2. 按 `F12` 或 `Cmd + Option + I` 打开开发者工具
3. 点击顶部 **Application** 标签
4. 左侧点击 **Cookies** → `https://xueqiu.com`
5. 找到 `xq_a_token`，复制 **Value** 列的值

### 2.2 更新 Cookie（两处都要更新）

**位置一：本地脚本**
```bash
# 编辑 scripts/crawl_xueqiu.py 第 28 行
COOKIE = "你的新Cookie值"
```

**位置二：GitHub Secrets**
```bash
# 更新 GitHub Secret
gh secret set XUEQIU_COOKIE --body "你的新Cookie值"
```

### 2.3 Cookie 维护
- Cookie 通常每月过期一次
- 过期后需要重新获取并更新
- 建议设置日历提醒，每月检查一次

## 三、自动更新机制

### 3.1 工作流程

```
┌─────────────────────────────────────────────────────────┐
│                     每周一凌晨 2:00                      │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│         GitHub Actions 自动启动                          │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│     从 GitHub Secrets 获取 Cookie                        │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│     运行爬虫脚本 (crawl_xueqiu.py)                       │
│     - 增量抓取新帖子                                     │
│     - 自动发现新提及的公司                                │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│     运行数据生成脚本 (generate_website_data.py)           │
│     - 更新 companyData.js                               │
│     - 更新 xueqiu_data.js                               │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│     提交代码到 GitHub                                    │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│     Vercel 自动检测到变化                                │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│     自动部署新网站                                       │
└─────────────────────────────────────────────────────────┘
```

### 3.2 关键文件

| 文件 | 作用 |
|------|------|
| `scripts/crawl_xueqiu.py` | 雪球爬虫主程序 |
| `scripts/generate_website_data.py` | 生成网站数据 |
| `raw/xueqiu/duanyongping_posts.json` | 爬取的帖子数据 |
| `raw/xueqiu/companies_config.json` | 公司配置 |
| `website/js/companyData.js` | 网站使用的公司数据 |

### 3.3 已知公司列表

爬虫内置了段永平常提及的公司：
- 腾讯、苹果、贵州茅台、网易、阿里巴巴、拼多多
- 谷歌、亚马逊、特斯拉、比亚迪、百度
- OPPO、vivo、步步高
- 万科、平安、京东、美的、格力、五粮液
- 可口可乐、微软、Meta 等

## 四、手动操作指南

### 4.1 首次配置（Cookie 已失效时）

```bash
# 1. 获取新 Cookie（见 2.1 节）

# 2. 更新本地脚本
# 编辑 scripts/crawl_xueqiu.py 第 28 行

# 3. 推送更新
git add scripts/crawl_xueqiu.py
git commit -m "更新雪球Cookie"
git push

# 4. 更新 GitHub Secret
gh secret set XUEQIU_COOKIE --body "新Cookie值"
```

### 4.2 本地测试爬虫

```bash
cd ~/Desktop/投资大佬agent知识库

# 测试全量爬取（首次使用或数据为空时）
python3 scripts/crawl_xueqiu.py --full

# 测试增量爬取（常规使用）
python3 scripts/crawl_xueqiu.py
```

### 4.3 本地生成网站数据

```bash
# 生成网站数据文件
python3 scripts/generate_website_data.py

# 更新到 website 目录
cp raw/xueqiu/posts_summary.json website/js/xueqiu_data.js
```

### 4.4 本地预览网站

```bash
cd ~/Desktop/投资大佬agent知识库/website
python3 -m http.server 3001
# 浏览器打开 http://localhost:3001
```

### 4.5 手动触发 GitHub Actions

1. 打开 https://github.com/amandapan1996/touzi-kb/actions
2. 点击 "雪球爬取与网站更新"
3. 点击 "Run workflow"
4. 选择模式：
   - `incremental`：增量更新（推荐）
   - `full`：全量爬取

## 五、GitHub Actions 配置

### 5.1 已配置的工作流

文件位置：`.github/workflows/crawl_and_deploy.yml`

- **触发时间**：每周一凌晨 2:00
- **手动触发**：支持
- **需要配置**：以下 GitHub Secrets

### 5.2 必需的配置

| Secret 名称 | 说明 | 如何获取 |
|------------|------|----------|
| `XUEQIU_COOKIE` | 雪球认证Cookie | 见 2.1 节 |
| `VERCEL_TOKEN` | Vercel API Token | vercel.com/account/tokens |
| `VERCEL_ORG_ID` | Vercel 组织ID | 项目 Settings |
| `VERCEL_PROJECT_ID` | Vercel 项目ID | 项目 Settings |

### 5.3 添加 Vercel 配置

```bash
# 添加 Vercel Token
gh secret set VERCEL_TOKEN --body "你的VercelToken"

# 添加 Vercel Org ID（需要从 Vercel 官网获取）
gh secret set VERCEL_ORG_ID --body "你的OrgId"

# 添加 Vercel Project ID（需要从 Vercel 官网获取）
gh secret set VERCEL_PROJECT_ID --body "你的ProjectId"
```

## 六、Vercel 部署

### 6.1 部署步骤

1. 注册 Vercel 账号（建议用邮箱注册）
2. 连接 GitHub 仓库 `amandapan1996/touzi-kb`
3. 选择 `website` 文件夹作为部署目录
4. 完成部署

### 6.2 项目配置

| 设置项 | 值 |
|--------|-----|
| Framework Preset | Other |
| Root Directory | `./` |
| Build Command | （留空）|
| Output Directory | `.` |

### 6.3 获取 Vercel 配置信息

1. 登录 https://vercel.com
2. 进入项目 → Settings → General
3. 找到 **Organization ID** 和 **Project ID**
4. 创建 API Token：Settings → Tokens → Create Token

## 七、故障排除

### 问题 1：Cookie 过期
**表现**：爬虫返回 401 错误或空数据
**解决**：
1. 重新登录雪球获取新 Cookie
2. 更新到本地文件和 GitHub Secrets

### 问题 2：GitHub Actions 失败
**解决**：
1. 查看 Actions 日志确定错误原因
2. 常见问题：Cookie 过期、Python 包缺失

### 问题 3：Vercel 未自动部署
**解决**：
1. 检查 GitHub 授权是否有效
2. 确认 Vercel 项目设置正确
